# Куб над Королевством

Локальный MVP-прототип по ГДД v0.3: idle-clicker/light strategy про осадную систему, которая постепенно разрушает огромный космический куб.

## Запуск

```bash
npm test
npm run balance
npm start
```

После запуска откройте `http://127.0.0.1:5173`.

## Что уже есть

- тап для добычи Королевских приказов;
- покупка и размещение орудий в слотах;
- 6 типов орудий с разными зонами поражения и скоростью;
- автоматическая стрельба, качество выстрела, критические попадания;
- HP куба, 5 слоёв, визуальные повреждения;
- падающие блоки и сбор Осколков Куба;
- улучшение орудий до 3 уровней;
- ручное наведение для точных орудий и слабых мест;
- камера по вертикальной поверхности куба;
- полный лабиринт прокачки на 30 узлов;
- локальное сохранение с миграцией версий и автономным прогрессом до 8 часов;
- ограниченное хранилище выпавших блоков без бесконечного роста DOM/сейва;
- полоса текущего слоя, объяснение уязвимых зон и причины заблокированного урона;
- подтверждение сброса прогресса и замены улучшенного орудия;
- звуковые эффекты и сохраняемые настройки звука;
- обучение первого сеанса;
- финальная катсцена и экран победы со статистикой;
- безопасный экспорт и импорт локального save.

## Баланс-симуляция

```bash
npm run balance
```

Скрипт берёт реальные числа из `src/gameState.js`, считает теоретический DPS по орудиям и прогоняет три профиля: пассивный, средний и оптимизатор. Отчёты сохраняются в:

- `output/balance/latest.json`
- `output/balance/latest.md`

В отчёте также есть solver-ячейка `Partitioned HP Suggestion`: она разделяет целевой бюджет времени на бой и накопление, считает `DPS_eff = HP / combat_time`, предлагает HP-вектор под боевой бюджет и отдельно показывает acquisition-wait, включая hard gate финального орудия. Acquisition остаётся диагностикой; release-gate учитывает обязательные total/combat guardrails и сравнение веток лабиринта.

Игра сбалансирована как combat-paced: бюджет времени живёт в разрушении куба, а не в ожидании покупок. Ключевые механики этого пейсинга:

- Королевская казна: единоразовая выплата за разрушение каждого слоя (`LAYER_REWARDS`), где награда за Внутреннее ядро покрывает ~3/4 стоимости осадной пушки;
- жалование инженера: каждый разрушенный слой навсегда увеличивает пассивную добычу приказов (`LAYER_ORDER_RATE_BONUS`);
- HUD-индикатор на этапе Сердца куба показывает, сколько ресурсов осталось до осадной пушки.

Текущий snapshot (HP-вектор из 4-й итерации солвера при 90% combat budget, применён в `CUBE_LAYERS`):

- пассивный профиль под присмотром: около 3ч 23м total, из них 3ч 22м combat, acquisition менее 2 минут;
- средний профиль: около 2ч 13м total;
- оптимизатор: около 1ч 36м total;
- total spread: около 2.11x; combat spread: около 2.10x; оба guardrails ok;
- hard gate осадной пушки: секунды вместо прежних 2ч 35м ожидания.

Шорткат 4x камнемёт L3 больше не проходит игру: урон гейтится достижимостью зоны слоя, а устаревшие орудия нужно заменять. Сейвы версионируются (`SAVE_VERSION`), старые сейвы v1 мигрируют с пропорциональным переносом прогресса слоя и ретроактивной выдачей наград за уже разрушенные слои.

## GitHub Pages

Репозиторий готов к публикации статической веб-версии через GitHub Pages. Workflow `.github/workflows/pages.yml` запускает `npm test`, собирает статические файлы в `dist/` и деплоит сайт через GitHub Actions.

После первого push включите в настройках репозитория:

`Settings -> Pages -> Build and deployment -> Source -> GitHub Actions`

## Не включено в этот MVP

- музыка и финальный набор звуков;
- финальный арт и детальная анимация рабочих;
- аккаунты, регистрация и серверная синхронизация сейвов;
- подтверждённый живыми плейтестами финальный баланс.
