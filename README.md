# Куб над Королевством

Локальный MVP-прототип по ГДД v0.3: idle-clicker/light strategy про осадную систему, которая постепенно разрушает огромный космический куб.

## Запуск

```bash
npm test
npm run balance
npm start
```

После запуска откройте `http://127.0.0.1:5173`.

## Что уже есть

- тап для добычи Королевских приказов;
- покупка и размещение орудий в слотах;
- 6 типов орудий с разными зонами поражения и скоростью;
- автоматическая стрельба, качество выстрела, критические попадания;
- HP куба, 5 слоёв, визуальные повреждения;
- падающие блоки и сбор Осколков Куба;
- улучшение орудий до 3 уровней;
- ручное наведение для точных орудий и слабых мест;
- камера по вертикальной поверхности куба;
- минимальный лабиринт прокачки на 10 узлов;
- локальное сохранение прогресса;
- экран победы со статистикой.

## Баланс-симуляция

```bash
npm run balance
```

Скрипт берёт реальные числа из `src/gameState.js`, считает теоретический DPS по орудиям и прогоняет три профиля: пассивный, средний и оптимизатор. Отчёты сохраняются в:

- `output/balance/latest.json`
- `output/balance/latest.md`

В отчёте также есть solver-ячейка `Partitioned HP Suggestion`: она разделяет целевой бюджет времени на бой и накопление, считает `DPS_eff = HP / combat_time`, предлагает HP-вектор под боевой бюджет и отдельно показывает acquisition-wait, включая hard gate финального орудия.

Текущий диагностический snapshot после разделения combat/acquisition:

- пассивный профиль: около 3ч 43м total, около 42м combat;
- средний профиль: около 3ч 59м total, около 30м combat;
- оптимизатор: около 1ч total, около 20м combat;
- total spread: около 4x;
- combat spread: около 2x;
- partitioned HP suggestion при 70% combat budget: около 1.68M total HP.

Шорткат 4x камнемёт L3 больше не проходит игру: урон гейтится достижимостью зоны слоя, а устаревшие орудия нужно заменять. Оставшийся риск баланса — распределение времени по слоям: ранние слои всё ещё проходят быстро, а сердце куба несёт слишком большую долю общего времени.

## GitHub Pages

Репозиторий готов к публикации статической веб-версии через GitHub Pages. Workflow `.github/workflows/pages.yml` запускает `npm test`, собирает статические файлы в `dist/` и деплоит сайт через GitHub Actions.

После первого push включите в настройках репозитория:

`Settings -> Pages -> Build and deployment -> Source -> GitHub Actions`

## Не включено в этот MVP

- финальная катсцена и музыка;
- 30-узловой полный лабиринт;
- полноценная система звуков;
- финальный арт и детальная анимация рабочих;
- точный баланс на 3-4 часа прохождения.
